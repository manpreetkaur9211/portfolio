The hardest habit to break when using AI agents isn't technical. It's this:

Stopping yourself from thinking in steps.

As developers, we're wired to think procedurally — do this, then this, then that. So when we use Claude Code, we instinctively prompt the same way. "Write a function that loops through X and returns Y."

That works. But it's not agentic.

The shift happened for me when I stopped describing *how* and started describing *what I want the world to look like*.

Instead of: "Write a filter function that removes jobs with 'Python' in the title"

I said: "Python roles are showing up — they're not relevant to my stack. Fix it."

Claude didn't just add a filter. It added the pattern constant, wired it into the pipeline, identified that old data in my Google Sheet was also affected, and cleaned that up too.

I described an outcome. It figured out the scope.

That's the actual power of agentic engineering — not that AI writes code faster, but that it can reason about what "done" really means.

The more you describe outcomes instead of steps, the more useful your agent becomes.

Are you prompting for tasks or outcomes?
